# Workflow: Spesifikasyondan Göreve

Amaç: Kod ajanlarının “doğru” ve “kontrollü” üretim yapabilmesi.

## Görev standardı
Her task şu alanları içermeli:
- Amaç
- Kapsam
- Done kriterleri
- Acceptance criteria referansı
- Data değişikliği var mı? (migration)
- Security etkisi var mı? (RLS/RBAC)
- Test gereksinimi
- Doküman güncelleme gereksinimi

## Örnek task tipleri
- Feature task (UI + API)
- Data task (migration + policy)
- Refactor task
- Bugfix task
- Performance task

## Çıktı
- Backlog listesi
- Sprint plan önerisi (MVP)
